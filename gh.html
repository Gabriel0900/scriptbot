Cresci no sítio, na zona rural, e mesmo com pouco acesso à tecnologia, esse sonho permaneceu. Quando entrei na EREM Fernando Bezerra e descobri as eletivas de robótica, vi a oportunidade de transformar aquele sonho em realidade.

Lição aprendida

O simples bem-feito funciona melhor que o complexo mal executado.

O que é o LEGO NXT?

Plataforma educacional da LEGO com foco em robótica e lógica de programação.

Por que é incrível para iniciantes?

Programação visual (blocos) facilita o aprendizado por representar funções complexas de forma simples.

Exemplo prático:

Para fazer um robô andar em código tradicional seriam várias linhas. No NXT, com 2 blocos (mover + loop), o robô já anda infinitamente.

Montagem do robô

Equilíbrio entre estrutura e código

Esqueleto

Estrutura sólida e equilibrada

Motores

Bem posicionados para subir rampas

Peso

Distribuição correta é essencial

Dicas essenciais:

  • Comece pelo esqueleto: estrutura sólida e equilibrada
  • Motores bem posicionados para subir rampas e vencer obstáculos
  • Atenção à tração e distribuição de peso
  • Inspiração: tanques, jipes ou carros como Tesla (base larga, comprimento suficiente)
  • Montagem mal feita = programação comprometida

Sensores essenciais

Sensor de Luz

  • Trabalha com luz refletida
  • Branco reflete muito (valor alto), preto absorve (valor baixo)
  • Problema: verde e preto têm valores parecidos → pode confundir o robô
  • Solução: ajustes manuais no código para diferenciar os dois (média, desvios, contexto)

Sensor Ultrassônico

  • Funciona como o "olho do robô"
  • Emite sinais de som e mede a distância por eco (como morcegos)
  • Usado para detectar obstáculos e desviar automaticamente
  • Precisão fundamental para navegação

Como programá-los?

No LEGO NXT, os sensores são configurados através de blocos específicos que interpretam seus valores e tomam decisões.

Se sensor_ultrassonico < 20 cm → virar para esquerda

Se sensor_luz < 30 → ajustar para direita

Essa lógica simples permite navegação autônoma em circuitos complexos.

Princípios da programação em blocos

Principais blocos:

Mover (Move)

Anda para frente ou para trás

Loop (Laço)

Repete comandos infinitamente

Switch (Condicional)

Toma decisões com base em sensores

Wait (Esperar)

Aguarda condições específicas

Personalização de sensores

Exemplo "ChatGPTIA"

// Lógica para seguir linha branca

Se sensor_luz < 30 → virar esquerda

Imagine que o sensor está lendo 40 (branco), 20 (verde/preto). O robô precisa seguir o branco. Se ele ler valor < 30, ele corrige para a esquerda, se for > 30, para a direita. Simples assim com blocos de decisão.

Por que programação em blocos é ideal para iniciantes?

  • Visualização clara do fluxo do programa
  • Redução de erros de sintaxe
  • Fácil compreensão de estruturas lógicas
  • Rápida prototipagem de ideias
  • Base sólida para aprender linguagens textuais depois

Erros comuns e lições aprendidas

Capricho é tudo

Montagem interfere diretamente na execução. Um robô mal montado não executa bem mesmo com o melhor código.

Teste antes da competição

Bugs acontecem nas horas mais críticas. Teste exaustivamente em condições similares às da competição.

Trabalho em equipe

Comunicação entre programador e montador é essencial. Ambos precisam entender as limitações e possibilidades.

Regra de ouro

"Não complique o simples." Às vezes a solução mais básica é a mais eficiente.

Minhas principais lições:

  1. Comece com projetos simples e vá aumentando a complexidade
  2. Documente tudo - configurações, códigos, ajustes
  3. Tenha um plano B (robô reserva, peças extras)
  4. O tempo de competição passa rápido - treine sob pressão
  5. Celebre cada pequeno progresso
