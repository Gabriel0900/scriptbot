O que é o LEGO NXT?

Plataforma Educacional

O LEGO Mindstorms NXT é uma plataforma revolucionária da LEGO voltada para educação em robótica, programação e engenharia. Combinando peças tradicionais da LEGO com sensores, motores e um bloco controlador programável, ele permite criar robôs inteligentes capazes de interagir com o ambiente.

Processador ARM7
256KB Flash
Conexão sem fio
Porta USB

Por que usar LEGO NXT?

  • Aprendizado prático de lógica de programação
  • Desenvolvimento de habilidades de engenharia
  • Estimula criatividade e resolução de problemas
  • Base para competições de robótica como a OBR

Como programamos com blocos?

Ambiente NXT-G

O software NXT-G utiliza uma abordagem visual de programação por blocos, onde cada bloco representa uma função ou comando específico. Os blocos são conectados para formar o fluxo do programa.

Vantagens

  • Intuitivo para iniciantes
  • Visualização clara do fluxo
  • Não requer conhecimento prévio de sintaxe
NXT-G - program.nxt
Start
Move [B+C] - Power: 75 - Duration: Unlimited
Loop
Switch [Light Sensor]
[Case 1] Move Left
[Case 2] Move Right

Principais Blocos de Programação

Move

Controla os motores para fazer o robô andar para frente, trás ou girar.

Loop

Repete um conjunto de comandos continuamente ou um número específico de vezes.

Switch

Toma decisões baseadas em entradas de sensores ou condições lógicas.

Sensor Blocks

Lê dados dos sensores (luz, toque, ultrassônico) para interação com o ambiente.

Erros e Acertos

Lições Aprendidas nas Competições

Robô Complexo Demais

Na primeira competição, desenvolvemos um robô com muitas funcionalidades e lógica complexa, que acabou falhando devido a:

  • Sobrecarga do processador (CPU do NXT)
  • Latência nas respostas aos sensores
  • Dificuldade para depurar problemas
Às vezes, menos é mais. A complexidade pode ser inimiga da confiabilidade.

Robô Simplificado

Na segunda competição, optamos por uma abordagem mais simples que resultou em:

  • Respostas mais rápidas aos sensores
  • Maior confiabilidade e menos falhas
  • Melhor pontuação nas provas
A simplicidade nos permitiu focar no essencial e executar com excelência.

Relação entre Montagem e Programação

Geometria do Robô

A distância entre as rodas e o posicionamento dos sensores afetam diretamente os parâmetros de programação.

Distribuição de Peso

Um robô desbalanceado requer ajustes na potência dos motores e pode afetar a precisão dos sensores.

Calibração de Sensores

A altura e ângulo dos sensores em relação à pista determinam os valores limiares na programação.

Principais Aprendizados

  • Testar cada componente isoladamente antes da integração
  • Documentar todas as alterações na montagem e programação
  • Manter versões de backup do programa funcional
  • Priorizar robustez sobre complexidade

Lógica Simples = Sucesso

Algoritmo para Seguir Linha

A solução mais eficaz nas competições foi um algoritmo simples de seguir linha baseado em valores limiares do sensor de luz.

Fluxograma

  • Início
  • Loop Infinito
  • Sensor de Luz < 30?
  • Sim → Virar à esquerda
  • Não → Virar à direita

Código em Blocos NXT-G

Start
  Move Forward (Motors B+C, Power 75)
  Loop Forever
    Switch (Light Sensor)
      Case 1 (Value < 30)
        Move Left (Motor C forward, Motor B backward)
      Case 2 (Value >= 30)
        Move Right (Motor B forward, Motor C backward)
    End Switch
  End Loop
End

Por que a simplicidade funciona?

Resposta Rápida

Menos processamento significa decisões mais rápidas, essencial para pistas com curvas fechadas.

Confiabilidade

Menos blocos = menos pontos de falha. O robô se recupera mais facilmente de erros.

Facilidade de Ajuste

Parâmetros como valores limiares e potência dos motores podem ser ajustados rapidamente.

Dica para Competições

Comece com a solução mais simples possível e só adicione complexidade se necessário. Na maioria dos casos, um algoritmo básico de seguir linha com ajustes finos é mais eficaz que soluções sofisticadas.

Nosso robô simplificado pontuou 50% a mais que a versão complexa

Missão para os futuros robóticos!

Desafio Inicial

Montar um robô estável

Base larga, sensores na altura correta e motores alinhados.

Programar com blocos simples

Comece com movimento básico e adicione sensores gradualmente.

Testar com pista

Use uma pista preta com curvas de diferentes raios para calibrar.

Passo a Passo Inicial

  1. 1 Montar chassi básico com 2 rodas motrizes e 1 roda livre
  2. 2 Posicionar sensor de luz ~1cm acima da pista, centralizado
  3. 3 Programar movimento para frente contínuo
  4. 4 Adicionar lógica de decisão baseada no sensor de luz
  5. 5 Testar e ajustar valores limiares e potência dos motores

Pista de Teste Recomendada

Uma pista de teste simples consiste em uma linha preta contínua sobre um fundo branco, com curvas suaves para testar a capacidade de seguir linha do robô.

Exemplo Simples: Seguidor de Linha com 4 Motores

Exemplo Simples: Seguidor de Linha (4 Motores)

Este exemplo mostra um programa básico de seguidor de linha usando 4 motores no LEGO NXT. O robô utiliza um loop contínuo e uma condição baseada no sensor de luz para decidir o movimento dos motores. Ideal para robôs maiores ou que precisam de mais força e estabilidade.

Lógica do Programa

  • Start
  • Loop Infinito
  • Se Sensor de Luz < 30:
  • Ative motores A e B para virar à esquerda (C e D parados)
  • Senão:
  • Ative motores C e D para virar à direita (A e B parados) 
Start
  Loop Forever
    If Light Sensor < 30 Then
      Motor A+B: Power 75 (esquerda)
      Motor C+D: Power 0
    Else
      Motor C+D: Power 75 (direita)
      Motor A+B: Power 0
    End If
  End Loop
End

Explicação: O robô lê o valor do sensor de luz. Se detectar linha preta (valor baixo), ativa os motores do lado esquerdo para corrigir a direção. Se detectar branco (valor alto), ativa os motores do lado direito. Isso mantém o robô seguindo a linha de forma simples e eficiente, mesmo com 4 motores.
